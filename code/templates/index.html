Aplicación demostrative del uso de CAS para autenticación

El siguiente enlace llevará al usuario a la pantalla de login del servidor CAS. La URL de ese enlace contiene el parámetro service que a su vez contiene la URL a la que debe regresar si el usuario consigue autenticarse con éxito. En este ejemplo, esa URL es la URL de esta misma aplicación, concretamente a la ruta /verify. Cuando el usuario regrese a esta URL, llevará como parámetro un ticket. La aplicación tomará ese ticket y lo canjeará en el servidor CAS. Si el ticket es válido, el servidor CAS nos responderá con información sobre el usuario. A partir de ese punto ya podemos considerar al usuario como legítimo.
